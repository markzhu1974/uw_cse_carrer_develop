路线3（**LLM / Agent Engineer 路线**）其实是2026-2028年最值得关注的一条路径。

但我要先纠正一个常见误区：

> 很多人以为 Agent Engineer = 调用 OpenAI API + LangChain。

实际上，大厂招聘中的 Agent 岗位已经越来越接近：

**ML Engineer + NLP Engineer + Software Engineer + AI Product Engineer**

的混合体。

从字节、腾讯混元、阿里通义、百度文心、Meta AI、微软 Copilot 的招聘要求来看，本科生最有机会进入的不是基础模型研究岗，而是：

* Agent Engineer
* AI Application Engineer
* AI Platform Engineer
* AI Coding Engineer
* Applied AI Engineer

这些岗位本质都属于路线3。

---

# 一、路线3的职业定位

先看AI岗位的金字塔。

```text
Research Scientist
       ↑
Applied Scientist
       ↑
Agent Engineer
       ↑
ML Engineer
       ↑

```

UW CE本科生最现实的位置：

```text
Agent Engineer
```

基础模型研究：

* ICML

而Agent方向：

* 更强调工程实现
* 更强调系统设计
* 更强调产品落地

这恰好符合CE背景。

# 二、Agent到底是什么？

Agent本质：

```text
LLM
+
Memory
+
Tool Use
+
Planning
+
Workflow
```

传统ChatGPT：

```text
Question
 ↓
LLM
 ↓
Answer
```

Agent：

```text
Question
 ↓
Planner
 ↓
Tools
 ↓
Memory
 ↓
Reflection
 ↓
Answer
```

例如：

用户：

```text
帮我分析Tesla财报并生成PPT
```

Agent：

```text
搜索财报
↓
提取数据
↓
分析趋势
↓
生成图表
↓
生成PPT
↓
输出结果
```

这就是Agent。

---

# 三、为什么最重视Agent

根据你前面统计的岗位：

Agent出现次数：

100+

已经超过：

* 推荐系统
* CV
* NLP

很多传统方向。

原因：

LLM能力越来越接近。

未来竞争点：

不是模型本身。

而是：

```text
谁能把模型变成生产力
```

Agent就是生产力层。
