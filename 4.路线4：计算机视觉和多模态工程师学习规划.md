路线4（**Computer Vision / Multimodal Engineer，计算机视觉与多模态方向**）其实是一个正在经历重大转型的赛道。

如果是2020年：

```text
CV = CNN + Detection + Segmentation
```

而到了2026年：

```text
CV = Vision + Language + Agent
```

所以现在如果UW CE学生要走CV路线，不能再按照传统的：

```text
OpenCV
→ CNN
→ YOLO
→ 找工作
```

的思路规划。

因为字节、腾讯、阿里、Meta、Google的新招聘需求已经变成：

* Vision-Language Model（VLM）
* Multimodal AI
* Embodied AI
* Robotics Vision
* Video Understanding

而不是单纯目标检测工程师。

---

# 一、CV方向的职业分层

首先要明确未来就业方向。

## 第一层：CV Engineer

典型岗位：

* 目标检测
* 图像分类
* OCR
* 工业视觉

工作内容：

```text
图像
↓
CNN/Transformer
↓
检测结果
```

---

## 第二层：Multimodal Engineer

典型岗位：

* GPT-4o
* Gemini
* Qwen-VL
* Claude Vision

工作内容：

```text
图像
+
文本
+
视频
↓
理解
```

---

## 第三层：Embodied AI

典型岗位：

* Tesla Optimus
* Figure AI
* Agility Robotics
* Physical Intelligence

工作内容：

```text
视觉
+
动作
+
决策
```

---

对于UW CE学生：

最适合的是：

```text
CV Engineer
↓
Multimodal Engineer
```

路线。

# 二、为什么CE学生做CV有优势

很多人认为CV是CS专业的天下。

其实不是。

CV系统最终落地：

```text
Camera
↓
Perception
↓
Model
↓
GPU
↓
Deployment
↓
Edge Device
```

这里涉及：

* Computer Architecture
* Embedded
* GPU
* Performance

这些正是CE优势。

---

例如：

### 自动驾驶

感知模型只是第一步。

真正难的是：

```text
100ms
↓
推理
↓
决策
```

实时性。

---

所以：

Meta

Tesla

NVIDIA

Apple

实际上非常喜欢：

```text
CV + System
```

背景的人。

# 九、2026-2028最有价值的CV路线

如果我是现在的大二UW CE学生，我不会把目标定为：

```text
传统CV工程师
```

因为岗位在缩减。

我会瞄准：

```text
Multimodal Engineer
```

能力结构：

```text
Computer Vision
          +
Transformer
          +
LLM
          +
Vision-Language
          +
System
```

具体课程组合建议：

| 学年   | 课程重点                        |
| ---- | --------------------------- |
| 大三秋  | CSE 421 + CSE 446 + CSE 455 |
| 大三冬  | CSE 447 + CV项目              |
| 大三暑假 | YOLO + ViT + 多模态项目 + 实习     |
| 大四   | 分布式系统 + GPU + 多模态工程         |

最终形成：

```text
YOLO
↓
ViT
↓
CLIP
↓
LLaVA/Qwen-VL
↓
Vision Agent
```

这条能力演进路线最符合未来3-5年字节、腾讯、Meta、Tesla、NVIDIA等公司对视觉AI人才的需求趋势。
